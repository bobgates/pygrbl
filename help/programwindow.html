If g-code files are loaded, using File-Open, they will appear in the program window. Execution can then be controlled using the play-stop and single step controls.

Pygrbl sends commands to fill grbls queue, but shows exactly which line of g-code is currently being worked on with a coloured highlight.

Pressing the stop button on the toolbar at any time will send an emergency stop to grbl, stopping milling immediately. The stop button is grey if grbl is not running, or red if it is currently executing a command, either in a program or manually.
